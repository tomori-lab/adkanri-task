## 📋 手動でやること

### 1. GitHub Pages を有効化

1. https://github.com/AXIS-AD/adkanri-task を開く
2. **Settings** → **Pages**
3. **Source**: Deploy from a branch
4. **Branch**: `master` / **Folder**: `/docs`
5. **Save**

数分後、https://AXIS-AD.github.io/adkanri-task/ で公開される

---

### 2. GAS の index.html を更新

リダイレクト機能を反映するため、GAS の index.html を差し替え：

1. [script.google.com](https://script.google.com/) でプロジェクトを開く
2. 左の `index.html` を開く
3. **このフォルダの `index.html`** の内容を全コピー
4. GAS の index.html の内容を全削除して貼り付け
5. **デプロイ** → **デプロイを管理** → **編集** → **バージョン: 新バージョン** → **デプロイ**

→ これで GAS の URL を直接開いても GitHub Pages に飛ぶようになる

---

### 3. task-dashboard を Render にデプロイ（タスク一覧を使う場合）

1. [Render](https://render.com) にログイン
2. **New** → **Web Service**
3. リポジトリ: `AXIS-AD/adkanri-task` を接続
4. 設定:
   - **Root Directory**: `task-dashboard`
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
5. **Environment** に追加（`.env.example` 参照）:
   - `CHATWORK_API_TOKEN`
   - `CHATWORK_MY_ACCOUNT_ID`
   - `CHATWORK_ROOM_1`, `CHATWORK_ROOM_2`
   - `CHATWORK_ROOM_AD` = `333632829`
   - `AD_MEMBER_IDS` = `10034061,9797164,5420288,10696465`
6. デプロイ完了後、`https://xxx.onrender.com` の URL をコピー

---

### 4. config.js に Render の URL を設定

1. `docs/config.js` を開く
2. `DASHBOARD_URL` に Render の URL を設定:
   ```js
   var DASHBOARD_URL = 'https://あなたのサービス名.onrender.com';
   ```
3. コミット & プッシュ
