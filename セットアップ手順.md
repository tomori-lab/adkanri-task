# TOアド管理依頼ツール セットアップ手順

## 概要

Google フォームの代替として、GAS Webアプリで依頼フォームを構築。
依頼 → スプシ記録 → チャットワークタスク化 を自動化。

### 新スプシ構造

| 列 | 内容 |
|---|---|
| A: ID | `REQ-20260305-0001` 形式 |
| B: タイムスタンプ | 送信日時 |
| C: 氏名 | 依頼者名 |
| D: 大分類 | 9カテゴリ |
| E: 小分類 | 大分類内の詳細分類 |
| F: 詳細 | JSON形式で全入力項目を格納 |
| G: ステータス | 未対応 / 対応中 / 完了 |

---

## セットアップ手順

### STEP 1: GASプロジェクト作成

1. スプレッドシート「アド管理依頼回答先」を開く
2. メニュー →「拡張機能」→「Apps Script」
3. 既存のコード（onFormSubmit）はそのまま残してOK（フォーム並行運用する場合）
4. ファイルを作成:
   - `Code.gs` の内容を**すべて**貼り付け（上書き）
   - 左の「+」→「HTML」→ ファイル名 `index` → `index.html` の内容を貼り付け

※ `SPREADSHEET_ID` は既に設定済み（既存スプシを使用）。別スプシを使う場合は `Code.gs` 5行目を変更。

※ 「氏名マスタ」シートは既に作成済み。A列=氏名、B列=ステータス（有効/無効）の形式で氏名を取得します。

### STEP 2: デプロイ

1. Apps Script エディタ →「デプロイ」→「新しいデプロイ」
2. 種類: 「ウェブアプリ」
3. 設定:
   - 説明: `TOアド管理依頼ツール v1.0`
   - 次のユーザーとして実行: `自分`
   - アクセスできるユーザー: `株式会社AXIS の全員`（またはリンクを知っている全員）
4. 「デプロイ」をクリック
5. 表示されるURLが依頼フォームのURL

### STEP 3: 動作確認

1. デプロイURLにアクセス
2. 氏名を選択 → 大分類 → 小分類 → 詳細入力 → 送信
3. 確認ポイント:
   - スプシの「依頼データ」シートにデータが追加されるか
   - チャットワークにタスクが作成されるか
   - 営業時間外の場合、全員にタスク化されるか

---

## 更新時の注意

- コード修正後は「デプロイ」→「デプロイを管理」→「編集（鉛筆アイコン）」→ バージョンを「新バージョン」にして更新
- 氏名の追加・削除は「氏名マスタ」シートで管理（GASを触る必要なし）
- チャットワークの通知先変更は `Code.gs` の `ASSIGN_MAP` を編集
