# デプロイ手順（統合アプリ）

## 構成

| コンポーネント | ホスティング | URL例 |
|---------------|-------------|-------|
| 統合アプリ（依頼+タスク） | Netlify | https://adkanri-task.netlify.app |
| 依頼フォーム | GAS（iframe埋め込み） | - |
| タスクダッシュボード | Render | https://xxx.onrender.com |

---

## 1. 統合アプリ（Netlify）

すでにデプロイ済みの場合はスキップ。

1. GitHub に `adkanri-task` をプッシュ
2. Netlify で `AXIS-AD/adkanri-task` を接続
3. ビルド設定:
   - 公開ディレクトリ: `docs`
   - ビルドコマンド: なし

---

## 2. タスクダッシュボード（Render）

### 2.1 リポジトリ構成

`task-dashboard` フォルダを Render にデプロイするため、以下のいずれか:

**A) adkanri-task リポジトリのルートを変更**
- Netlify: `docs` を公開
- Render: 別サービスで `task-dashboard` を指定（サブディレクトリ対応が必要）

**B) task-dashboard を別リポジトリにする（推奨）**
- `task-dashboard` を単独の GitHub リポジトリにコピー
- Render でそのリポジトリをデプロイ

**C) モノレポで Render にサブパス指定**
- Render は Root Directory で `task-dashboard` を指定可能

### 2.2 Render での手順

1. [Render](https://render.com) にログイン
2. **New** → **Web Service**
3. リポジトリを接続（adkanri-task または task-dashboard 単独）
4. 設定:
   - **Root Directory**: `task-dashboard`（adkanri-task の場合）
   - **Build Command**: `npm install`
   - **Start Command**: `npm start`
   - **Instance Type**: Free
5. **Environment** に以下を追加:
   - `CHATWORK_API_TOKEN`
   - `CHATWORK_MY_ACCOUNT_ID`
   - `CHATWORK_ROOM_1`
   - `CHATWORK_ROOM_2`
   - `CHATWORK_ROOM_AD` = `333632829`
   - `AD_MEMBER_IDS` = `10034061,9797164,5420288,10696465`
6. **Create Web Service**
7. デプロイ完了後、`https://xxx.onrender.com` のURLをコピー

### 2.3 統合アプリにURLを設定

1. `docs/config.js` を開く
2. `DASHBOARD_URL` に Render のURLを設定:
   ```js
   var DASHBOARD_URL = 'https://adkanri-task-dashboard.onrender.com';
   ```
3. コミットして Netlify に再デプロイ

---

## 3. 動作確認

1. https://adkanri-task.netlify.app を開く
2. **依頼する** タブ: GAS フォームが表示されること
3. **タスク一覧** タブ:
   - **全タスク（アド管理）**: アド管理ルームの全タスク
   - **自分の依頼**: メールアドレス入力 → 自分が依頼したタスクのみ

---

## 注意事項

- **Render Free プラン**: 15分間アクセスがないとスリープする。初回アクセスは数十秒かかる場合あり
- **CORS**: task-dashboard は同一オリジンで iframe 表示するため、CORS 設定は不要
- **機密情報**: `.env` は Git に含めない（.gitignore 済み）
